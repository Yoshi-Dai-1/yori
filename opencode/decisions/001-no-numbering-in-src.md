# ADR 001: プロダクションコードのディレクトリにナンバリングを使わない

**日付**：2025年  
**状態**：採用

---

## 状況

フォルダの先頭に `01_auth/`、`02_stock/` のようにナンバーを振る手法を検討した。
ファイルエクスプローラー上での表示順序を制御できる点が魅力だった。

## 決定

プロダクションコード（`src/` 以下）のディレクトリにはナンバリングを使わない。
ドキュメント・教材（`docs/`・教材リポジトリ）には例外として使用を許可する。

## 理由

1. **importパスの破壊**：`03_auth/` を `04_auth/` に変更すると、そのパスを参照するすべてのimport文が壊れる
2. **git差分の汚染**：順序変更のたびに複数ファイルの rename が発生し、レビューが困難になる
3. **AIのパス誤認識**：ナンバーは意味情報を持たないため、AIがパスを補完する際にノイズになる
4. **アーキテクチャが順序を表現すべき**：`app → pages → features → shared` の依存順序は `ARCHITECTURE.md` に明記すれば十分であり、ディレクトリ名に埋め込む必要はない

## トレードオフ

- ファイルエクスプローラーでの表示順序は制御できなくなる
- ただしIDEの検索（Cmd+P等）を使えば順序より名前で探す方が速い

## 例外

`docs/` や教材リポジトリでは「読む順序が意味を持つ」ため、ナンバリングを許可する。
→ 詳細は `architectures/document-project.md` を参照
