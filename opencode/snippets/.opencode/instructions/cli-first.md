# CLI First

AI エージェントが外部リソース（コード・環境・データ・CI状態等）にアクセスする際は、常に CLI 経由を最優先する。このルールは全セッション・全フェーズで有効。

## 判断基準

- 「まず人間に聞く」ではなく「まず CLI を試す」がデフォルト動作
- CLI のインストール・代替CLIの調査・webfetch による公式ドキュメント確認を経て、すべての CLI 手段が尽きた場合のみ、人間に支援を依頼する

## 手順

1. `command -v <tool>` で CLI の存在確認
   - 存在する → `--help` で使い方を確認して実行
   - 存在しない → 次のステップへ
2. `webfetch` で以下を調査する
   - その CLI のインストール方法
   - 代替となる CLI の有無
   - 公式ドキュメント上の正しいツール名・パス
3. 調査結果に基づいて行動する
   - インストール可能 → インストールを試行し、再度 `command -v` で確認
   - 代替CLIがある → 代替を試行
   - 情報不足 → さらに調査
4. すべての CLI 手段が尽きた場合のみ、人間に依頼する

## 適用範囲

- コードの取得・確認（Git, clasp, gh, curl 等）
- 環境のセットアップ
- データの確認・取得
- デプロイ・CI/CD 状態の確認

上記は例示であり、未記載のサービス・ツールにもこの手順を適用する。
