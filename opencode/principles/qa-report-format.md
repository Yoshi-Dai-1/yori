# QA評価レポート形式

@evaluator が評価結果を報告する際のテンプレート。

## テンプレート

```markdown
## QA評価レポート [スプリント番号]

### 判定：[PASS / FAIL]
### 評価方法：[playwright-cli による実機確認 / コードレビューベース（フォールバック）]

### 基準別評価

| 基準 | 評価 | 根拠 |
|------|------|------|
| Functionality | [PASS/FAIL] [%] | [確認した内容] |
| Code Quality | [PASS/FAIL] | [確認した内容] |
| Design Quality | [PASS/FAIL] | [スクリーンショット: .playwright-cli/xxx.png] |
| Originality | [PASS/FAIL] | [具体的なパターン名・判断根拠] |
| Craft | [PASS/FAIL] | [確認した内容] |
| Stability | [PASS/FAIL] | [確認した内容] |
| NFR | [PASS/FAIL/－] | [定義されていない場合は「－（測定のみ）: 計測値を記載」] |

### 発見した問題（FAILの場合）

[重要度: HIGH/MEDIUM/LOW]
場所：[ファイル名:行番号 またはUI上の操作手順]
問題：[何が問題か]
期待：[正しい動作]

### 次のスプリントへの申し送り

[実装エージェントが次のスプリントで注意すべき点]

### 総評

[全体的な品質の評価。良い点と改善点を具体的に記述]
```

## 使い方

@evaluator は評価完了後、このテンプレートに従ってレポートを出力する。
各フィールドは省略しない。該当しない場合は「なし」と明記する。
