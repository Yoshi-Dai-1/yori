# Plugin Events: session.deleted の代替案

`handoff.ts` は `session.deleted` イベントでセッション終了を検知する設計だが、
このイベントがファイル書き込み可能なタイミングで発火するか未確認。

## 検証方法

1. `session.deleted` で handoff が生成されるか確認する
2. 生成されない場合、以下を順に試す

## 代替案（試行順）

### 案A: `session.idle`

セッションがアイドル状態になったときに発火する。
通知プラグインの公式サンプルで使用されている実績あり。

**問題点**: メッセージごとに発火するため、重複防止が必要。
`HANDOFF_FILLED` マーカーですでに重複防止は実装済みだが、
セッション途中のアイドルでもマーカーなし状態で更新してしまう。

**対策**: handoff 生成の条件を「編集中のファイルがない」
（= git diff --name-only が空）場合に限定することで、
セッション終了時のみ生成するよう近似できる。

### 案B: `session.updated`

`session.created` / `session.deleted` 以外のすべての更新で発火。
`session.idle` よりは頻度が低い可能性があるが、まだ発火頻度は高い。

**問題点**: 案Aと同じく発火頻度が高い。
**対策**: 発火間隔を制御する仕組み（前回生成からN分以上経過）を併用する。

### 案C: 定期チェック（cron 相当）

`experimental.session.compacting` イベントがコンテクスト圧縮時に発火する。
セッションが長期間続いて圧縮が発生したタイミングで handoff を生成する。

**問題点**: 短いセッションでは圧縮が発生せず handoff が生成されない。
**対策**: 圧縮 + セッション終了の両方で生成する。

## 決定

初回リリースは `session.deleted` で実装する。
動作確認後、上記代替案へのフォールバックを検討する。
