## 月次レビュー手順（毎月1回）

AIへの指示：
```
.opencode/rules/ の各ルールファイルを読んで、以下を確認してください：

1. 各ルールのpathsパターンに一致するファイルが
   先月編集されたか確認する（git logで確認）
2. 一度も発火していないルールをリストアップする
3. 重複しているルールを特定する

報告形式：
## ルール参照レポート [YYYY-MM]

### 発火確認済み
- rule-name: paths対象ファイルが[N]回編集された

### 発火なし（削除候補）
- rule-name: pathsパターンに一致するファイルが編集されなかった

### 統合候補
- rule-a と rule-b: 内容が重複している
```

## ルール健全性チェック

新しいルールを追加する前に確認する：
- 同じ内容のルールが既に存在しないか
- pathsパターンが適切か（広すぎ・狭すぎ）
- 「判断基準（Why）」が書かれているか
